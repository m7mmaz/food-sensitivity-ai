# Food Sensitivity App - Gemini + Vercel

هذه النسخة جاهزة للرفع إلى GitHub ثم النشر على **Vercel** حتى يعمل:
- البحث المحلي داخل التطبيق
- البحث الأونلاين عن المنتجات والمكونات
- تحليل ذكي باستخدام **Google AI (Gemini)**
- عرض مصادر النتيجة داخل التطبيق
- حد يومي للبحث الأونلاين على نفس الجهاز عبر `localStorage`

## مهم جداً قبل الرفع
لا تضع مفتاح Gemini داخل الملفات أو داخل GitHub.
المفتاح يجب أن يضاف داخل **Vercel Environment Variables** فقط.

## متغير البيئة المطلوب
- `GEMINI_API_KEY` = مفتاح Google AI Studio API

## خطوات النشر
1. ارفع جميع الملفات إلى Repository على GitHub.
2. افتح Vercel وسجّل الدخول بحساب GitHub.
3. اختر **Import Project** ثم اختر الريبو.
4. داخل إعدادات المشروع في Vercel أضف متغير البيئة:
   - `GEMINI_API_KEY`
5. انشر المشروع.

## الحد اليومي
الحد اليومي مضبوط حالياً على **25** عملية بحث أونلاين لكل جهاز/متصفح في اليوم.
لتعديله:
- افتح `config.js`
- عدل القيمة:
  - `dailySearchLimit: 25`

مهم: هذا الحد محفوظ على نفس الجهاز عبر `localStorage`.
إذا كنت تريد حدًا صارمًا ومركزيًا لكل المستخدمين، ستحتاج قاعدة بيانات أو KV مثل Vercel KV.

## الملف المسؤول عن التحليل
- `api/analyze.js`

يقوم بـ:
- استقبال اسم المنتج من الواجهة
- إرسال الطلب إلى Gemini API
- استخدام Google Search داخل Gemini
- إرجاع نتيجة عربية منظمة مع مصادر

## ملاحظات مهمة
- النتيجة الذكية لا تُغني عن قراءة ملصق العبوة الحالي.
- بعض المنتجات تتغير تركيبتها حسب البلد أو وقت التصنيع.
- عند ضعف الثقة أو نقص المصادر، يعرض النظام: `يحتاج تحقق`.
